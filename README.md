# MLOps HW08 — ML System, Monitoring, Data Quality & Architecture

## Описание проекта

Данный проект реализует набор практических заданий по развертыванию ML-системы, мониторингу, анализу качества данных и проектированию архитектуры для задачи виртуального product placement.

---

## Задание 1 — Дерево метрик ML-системы

Разработана система метрик, связывающая бизнес-цели, ML-метрики и технические ограничения инфраструктуры. Основная цель — увеличение выручки онлайн-сервиса через улучшение качества рекомендаций и стабильность системы при высокой нагрузке. Были определены ключевые бизнес-метрики (конверсия, LTV), ML-метрики (precision@k, coverage, online A/B метрики) и технические SRE-ограничения (latency, RPS, availability). Также зафиксированы архитектурные компромиссы в виде ADR-подхода.

---

## Задание 2 — Мониторинг (Prometheus, Grafana, MLflow)

Выбрана метрика latency модели инференса как основная для мониторинга. Реализован экспорт метрик в Prometheus через /metrics endpoint, включая кастомную histogram метрику задержки инференса. Grafana используется для визуализации latency и системных метрик, Prometheus — для сбора, MLflow — для трекинга экспериментов. Вся инфраструктура развернута через Docker.

---

## Задание 3 — Дрифт и деградация модели

Смоделирован data drift путём изменения входного распределения данных между reference batch и current batch. Разница в распределении приводит к изменению статистик и фиксируется системой мониторинга. Используется подход сравнения батчей данных для выявления деградации качества модели и изменения поведения предсказаний.

---

## Задание 4 — Data Quality Ops (DQOps)

Настроена система контроля качества данных с использованием DQOps.

---

## Задание 5 — Архитектура Virtual Product Placement

Разработана потоковая архитектура (Kappa-like) для обработки видеоданных в реальном времени. Используется Redis как брокер сообщений для передачи кадров между компонентами пайплайна. YOLO выполняет детекцию людей, далее происходит сегментация и наложение визуальных элементов, после чего генеративная модель отвечает за реалистичную интеграцию бренда в изображение.

MLflow используется для трекинга обучаемых компонентов (детектор и генеративная модель), включая версионирование моделей и метрик качества. Система мониторится через Prometheus и Grafana, инфраструктура контейнеризирована с использованием Docker.
